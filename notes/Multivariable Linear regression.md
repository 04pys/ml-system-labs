# 다중 선형 회귀 복습 정리

## 1. 다중 선형 회귀의 핵심

앞서 배운 선형 회귀에서 입력 변수 `x`가 1개였다면, 다중 선형 회귀는 **입력 변수 `x`가 여러 개일 때의 선형 회귀**다. 즉, 하나의 `y`를 예측할 때 여러 feature를 함께 사용하는 형태다. 위키독스 예제에서는 `x1`, `x2`, `x3` 세 개의 퀴즈 점수로 최종 점수 `y`를 예측한다.

기본 가설은 다음과 같다.

H(x) = w1x1 + w2x2 + w3x3 + b

여기서

- `w1, w2, w3`: 각 feature에 대응하는 가중치
- `b`: 편향(bias)

즉, 각 입력이 결과에 얼마나 영향을 주는지를 가중치로 조절하고, 마지막에 bias를 더해 전체 예측값을 만든다.

---

## 2. sample과 feature 감각 잡기

다중 선형 회귀에서는 보통 데이터를 **행렬 X**로 관리한다.

- 행(row) = sample 수
- 열(column) = feature 수

예를 들어,

- 훈련 데이터(sample)가 5개
- 각 데이터를 설명하는 feature가 3개

라면 `X`는 **5 x 3**, `y`는 **5 x 1** 형태로 생각하면 된다.

즉,

- `X`: sample x feature
- `y`: sample x 1

이 관점을 잡아두면 입력 변수가 많아져도 데이터를 훨씬 체계적으로 이해할 수 있다. 위키독스의 예제도 5개의 데이터와 3개의 입력값으로 구성되어 있다.

---

## 3. 왜 행렬과 벡터로 표현하는가

feature가 1개일 때는 `Wx + b`로 간단히 볼 수 있지만, feature가 많아지면 `w1x1 + w2x2 + w3x3 + ...`처럼 식이 길어진다. feature 수가 커질수록 변수도 많이 선언해야 하고, 가설식도 복잡해진다.

그래서 다중 선형 회귀에서는 보통 다음처럼 생각한다.

- `X`: 입력 데이터 행렬
- `W`: 가중치 벡터
- `b`: 편향

그러면 전체 가설을 다음처럼 한 번에 표현할 수 있다.

H(X) = XW + b

이 표현의 장점은 다음과 같다.

- feature가 많아져도 식이 간단함
- 여러 변수를 일일이 선언하지 않아도 됨
- 행렬 연산으로 한 번에 계산 가능함
- 여러 sample에 대한 예측을 동시에 처리할 수 있음

즉, 다중 선형 회귀에서는 **“여러 변수의 회귀를 벡터/행렬 연산으로 압축해서 다룬다”**는 감각이 중요하다.

---

## 4. `W`와 `b`의 shape 이해

다중 선형 회귀에서 shape 감각은 매우 중요하다.

예를 들어 `X`가 `5 x 3`이라면, 각 sample마다 feature가 3개 있다는 뜻이다.  
그러면 각 feature에 대응하는 weight도 3개 필요하므로 `W`는 `3 x 1`이 된다.

이유는 다음과 같다.

- `X`: `(5 x 3)`
- `W`: `(3 x 1)`

그래야 행렬곱이 가능하고, 결과는

- `(5 x 3) @ (3 x 1) = (5 x 1)`

이 된다.

이 `(5 x 1)` 결과가 각 sample에 대한 예측값이다.

즉,

- sample 수는 `X`의 행 수에 담기고
- feature 수는 `X`의 열 수와 `W`의 행 수에 맞춰지며
- 최종 출력은 각 sample마다 예측값 하나이므로 `(sample 수 x 1)`이 된다

이 shape 흐름을 머릿속에 그릴 수 있어야 한다.

---

## 5. `b`는 왜 하나만 있어도 되는가

편향 `b`는 보통 1개의 값으로 둔다.  
그런데 예측값 `XW`는 `(sample 수 x 1)` 형태다. 이때도 문제없는 이유는 **broadcasting** 때문이다.

즉, `b`는 계산 과정에서 자동으로 sample 수에 맞게 확장되어 각 예측값에 더해진다.

직관적으로 보면,

- `W`는 각 feature의 영향력을 조절하고
- `b`는 전체 예측값을 공통으로 위아래 이동시키는 역할을 한다

그래서 `b`를 sample 수만큼 직접 만들 필요가 없다.  
**bias는 하나지만, 모든 sample의 예측값에 공통으로 더해진다**고 이해하면 된다.

이 부분은 단순 선형 회귀와 다중 선형 회귀 모두에서 중요한 감각이다.

---

## 6. 비용 함수: 여전히 MSE 사용

다중 선형 회귀에서도 손실 함수는 보통 **평균 제곱 오차(MSE)** 를 사용한다.

개념은 선형 회귀와 완전히 같다.

- 예측값과 실제값의 차이를 구하고
- 그 차이를 제곱해서 음수/양수 상쇄를 막고
- 전체 평균을 구해 손실로 사용한다

즉, 목표는 결국 **예측값과 실제값의 차이를 최소화하는 `W`와 `b`를 찾는 것**이다.

정리하면,

cost(W, b) = (1/n) * Σ[ y(i) - H(x(i)) ]^2

다중 선형 회귀라고 해서 비용 함수의 본질이 달라지는 것은 아니다. 달라지는 것은 입력이 여러 개라는 점뿐이고, 학습 목표 자체는 여전히 MSE를 최소화하는 것이다.

---

## 7. 학습의 핵심 흐름

다중 선형 회귀의 학습 흐름도 선형 회귀와 본질적으로 동일하다.

1. `hypothesis` 계산
2. `cost` 계산
3. gradient 초기화
4. 역전파로 gradient 계산
5. optimizer가 파라미터 업데이트

즉, 단순 선형 회귀에서 하던 학습 루프가 그대로 유지되며, 달라지는 것은 입력이 스칼라가 아니라 여러 feature를 가진 벡터/행렬이라는 점이다.

중요한 직관은 다음과 같다.

- `W`의 각 원소는 각 feature의 영향력을 조정
- `b`는 전체 예측값을 평행이동
- 학습을 반복할수록 `W`, `b`가 손실이 줄어드는 방향으로 바뀜

즉, 다중 선형 회귀는 “원리는 같고, 입력 구조만 확장된 선형 회귀”라고 보면 된다.

---

## 8. 예측 시 `with torch.no_grad():`를 쓰는 이유

실습 메모에서 꼭 기억해야 할 부분이다.

`with torch.no_grad():` 블록 안에서 수행되는 연산은 **기울기 계산을 하지 않는다**.  
즉, PyTorch가 연산 그래프를 만들지 않고, 미분을 위한 추적도 하지 않는다.

왜 이렇게 하냐면, **예측 단계에서는 gradient가 필요 없기 때문**이다.

학습 단계에서는

- 손실을 계산하고
- `W`, `b`를 업데이트해야 하므로
- gradient 추적이 필요하다

하지만 예측 단계에서는 이미 학습된 `W`, `b`로 값만 계산하면 된다.  
이때는 gradient를 구할 필요가 없으므로 `torch.no_grad()`를 사용하면 된다.

정리하면,

- **학습(training)**: gradient 필요
- **예측(inference)**: gradient 불필요

그래서 예측할 때 `with torch.no_grad():`를 쓰는 것이다.

---

## 9. 단순 선형 회귀와의 차이

단순 선형 회귀와 다중 선형 회귀의 가장 큰 차이는 입력 변수의 개수다.

### 단순 선형 회귀
- 입력 `x`가 1개
- 가설: `H(x) = Wx + b`

### 다중 선형 회귀
- 입력 `x`가 여러 개
- 가설: `H(x) = w1x1 + w2x2 + ... + b`
- 또는 행렬 형태로 `H(X) = XW + b`

하지만 핵심 원리는 같다.

- 둘 다 선형 모델
- 둘 다 MSE를 줄이는 방향으로 학습
- 둘 다 gradient descent를 사용
- 둘 다 `W`, `b`를 학습

즉, 다중 선형 회귀는 선형 회귀의 개념을 여러 입력으로 확장한 것이다.

---

## 10. 빠르게 떠올려야 할 복습 트리거

### 다중 선형 회귀란
- 입력 변수 `x`가 여러 개일 때의 선형 회귀
- 여러 feature로 하나의 `y`를 예측

### 데이터 shape
- `X`: `(sample 수 x feature 수)`
- `y`: `(sample 수 x 1)`

### 파라미터 shape
- `W`: `(feature 수 x 1)`
- `b`: 1개 값

### 핵심 가설
- `H(X) = XW + b`

### `b`가 하나여도 되는 이유
- broadcasting으로 전체 예측값에 자동 적용됨

### 비용 함수
- MSE 사용
- 예측값과 실제값 차이의 제곱 평균

### 학습의 본질
- `W`, `b`를 cost가 줄어드는 방향으로 업데이트

### 예측 단계
- `with torch.no_grad():` 사용
- gradient 계산 끔

---

## 11. 한 번에 기억할 요약

다중 선형 회귀는 여러 feature를 사용해 하나의 값을 예측하는 선형 회귀다. 데이터는 보통 `X`를 `(sample 수 x feature 수)` 행렬로, `y`를 `(sample 수 x 1)` 형태로 두고 다룬다. 이때 가설은 `H(X) = XW + b`로 표현할 수 있으며, `W`는 각 feature의 영향력을 나타내고 `b`는 전체 예측값을 공통으로 이동시키는 편향이다. `b`는 broadcasting 덕분에 하나의 값만 있어도 전체 예측에 적용된다. 학습 목표는 여전히 MSE를 최소화하는 것이며, 예측 단계에서는 gradient 계산이 필요 없으므로 `with torch.no_grad():`를 사용한다.
